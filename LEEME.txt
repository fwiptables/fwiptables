################################################## La primera opción:

#####         [Lanzar guia]

  system-info: Muestra la informacion del sistema
     net-info: Muestra informacion de la red
  full-wizard: Asistente y lanza completo cortafuegos
  mini-wizard: Asistente y lanza simple cortafuegos
          cli: Interfaz de linea de comandos con menus
          gui: Interfaz grafico de usuario

#####         [Lanza reglas de iptables]

   eraserules: Borra las reglas de iptables de ipv4 y de ipv6
      offline: Desconecta el cortafuegos de todo excepto localhost
 allpermisive: Configura el cortafuegos muy permisivo
       custom: Configura el cortafuegos especificando un archivo.cfg
[archivo.cfg]: Configura el cortafuegos con un archivo.cfg

#####         [Lista las reglas de tablas de iptables]

               La tabla de filtros con nombres
         list: Lista las reglas de ipv4 de filtros de entrada y salida
      forward: Lista las reglas de ipv4 de filtros de reenvio
        list6: Lista las reglas de ipv6 de filtros de entrada y salida
     forward6: Lista las reglas de ipv6 de filtros de reenvio

               La tabla de filtros con numeros
      listnum: Lista las reglas de ipv4 de filtros de entrada y salida en numeros
   forwardnum: Lista las reglas de ipv4 de filtros de reenvio en numeros
     list6num: Lista las reglas de ipv6 de filtros de entrada y salida en numeros
  forward6num: Lista las reglas de ipv6 de filtros de reenvio en numeros

#####         [Las otras tablas de iptables]

      nat-ip4: Lista las reglas de ipv4 de nat iptables
      nat-ip6: Lista las reglas de ipv6 de nat iptables
   mangle-ip4: Lista las reglas de ipv4 de mangle iptables
   mangle-ip6: Lista las reglas de ipv6 de mangle iptables
      raw-ip4: Lista las reglas de ipv4 de raw iptables
      raw-ip6: Lista las reglas de ipv6 de raw iptables
 security-ip4: lista las reglas de ip4 de security iptables
 security-ip6: Lista las reglas de ip6 de security iptables
     ebtables: Lista las reglas de nft de ethernet
    arptables: Lista las reglas de arp arptables

#####         [Modifica config]

       new-full-config: Crea una nueva configuracion.cfg en ingles
 nueva-completa-config: Crea una nueva configuracion.cfg en español
       new-mini-config: Crea una nueva basica configuracion.cfg en ingles
     nueva-mini-config: Crea una nueva basica configuracion.cfg en español
         modify-config: Edita una existente configuracion.cfg
           show-config: Muestra una existente configuracion.cfg
            del-config: Borra unaa existente configuracion.cfg

#####         [Lista las conexiones abiertas]

        route: Muestra los dispositivos de red y la tabla de rutas
           ss: Muestra las conexiones con comando ss
      netstat: Muestra las conexiones con comando netstat
         lsof: Muestra las conexiones con command lsof
  ipforwading: Muestra la configuracion del kernel sobre reenvios ip
      tcpdump: Escuchar las conexiones ip
          arp: Lista las ultimas conexiones arp

#####         [Recuperador de firewall en formato standard]

      fw-list: Lista los archivos de firewall
      fw-save: Guarda los archivos de firewall
   fw-restore: Indica como restaurar los archivos de firewall

#####         [Herramienta icmp]

     ping-ip4: Hace un ping a una host con ipv4 icmp
     ping-ip6: Hace un ping a una host con ipv6 icmp

#####         [Herramienta tcp]

    trace-ip4: Traza ruta a una host con ipv4 tcp
    trace-ip6: Traza ruta a una host con ipv6 tcp
         host: Resuelve la ip de un host
    resolveip: Resuelve la ip de un host y viceversa
       geo-ip: Localiza una ip o host

#####         [Opciones de autorizacion]

    auth-sudo: Permite autorizacion a fwiptables a usuarios elegidos
  remove-sudo: Borra la autorizacion otorgada con fwiptables
    list-sudo: Lista autorizacion a fwiptables de usuarios elegidos

#####         [Version]

      version: Informacion de la actual version fwiptables
      upgrade: Actualiza fwiptables a la ultima version
 dev-versions: Version en desarrollo de fwiptables
 all-versions: Todas las versiones de fwiptables

#####         [Show etc]

        hosts: muestra /etc/hosts
   protocolos: muestra /etc/protocols
     services: muestra /etc/services
  hosts.allow: muestra /etc/hosts.allow
   hosts.deny: muestra /etc/hosts.deny

#####         [Show log]

        hosts: busca los logs de sudo fwiptables para /var/log
   protocolos: busca log logs de sshd para /var/log
     services: busca los logs dando un patron para /var/log
     services: busca los logs recursivos dando un patron para /var/log

#####         [Ayuda]

        About: Acerca de fwiptables
      Depends: Muestra las dependencias de fwiptables
     examples: Regenerar ejemplos de fwiptables
         help: Muestra la ayuda en ingles
        ayuda: Muestra la ayuda en español

################################################## La segunda opcion (opcional)

    si lanzas sin especificar una config tomara por defecto default-full-english.cfg
    Puedes modificar y crear archivos en la carpeta /etc/fwiptables

################################################## Ejemplos

   fwiptables custom games-example1.cfg    [lanza un firewall modificado]
   fwiptables eraserules                   [Borra las reglas de iptables] 
   fwiptables list                         [Lista las reglas de iptables] 
   fwiptables help                         [Muestra la ayuda en ingles] 

##################################################  Ejemplos de las variables de configuración

# Fran FireWall iptables from fwiptables config file
# _ _ _ _ _ _ # INICIO .......... Opciones Necesarias .......... .......... #
# # # _ _ _ _ _ _ # el iptables firewall netfilter, elige uno o dos
allow_legacy= # _ _ _ _ _ _ # lanza xtables, vacio para si, o escribe no
allow_nft=no # _ _ _ _ _ _ # lanza neftables, vacio para si, o escribe no
# # # _ _ _ _ _ _ # procolo ip, modificar con vacio o no
allow_ipv4= # _ _ _ _ _ _ # varcio para ejecutar el firewall con ipv4 o no para no
allow_ipv6=no # _ _ _ _ _ _ # varcio para ejecutar el firewall con ipv6 o no para no
# # # _ _ _ _ _ _ # puertos Cliente, añadir con ',' y poner rangos con ':'
client_tcp=http,https,ssh,git # _ _ _ _ _ _ # puertos tcp para ser cliente
client_udp=domain,bootpc,ntp,https # _ _ _ _ _ _ # puertos udp para ser cliente
# # # _ _ _ _ _ _ # Puertos Servidor, añadir con ',' y poner rangos con ':'
server_tcp= # _ _ _ _ _ _ # puertos tcp para servidor
server_udp= # _ _ _ _ _ _ # puertos udp para servidor
logserver_tcp= # _ _ _ _ _ _ # puertos tcp para logear servidor
logserver_udp= # _ _ _ _ _ _ # puertos udp para logear servidor
# # # _ _ _ _ _ _ # Estado de entrada selectivo
input_state='-m state --state new,related,established' # _ _ _ _ _ _ # modificar con vacio o -m state --state new,related,established,untracked,invalid
# # # _ _ _ _ _ _ # vacio para funcionar con reglas de salida solamente or no
only_output= # _ _ _ _ _ _ # vacio, reglas solo para salida o no
# # # _ _ _ _ _ _ # FINAL .......... Opciones Necesarias .......... .......... #
# # # _ _ _ _ _ _ # INICIO .......... Opciones opcionales .......... .......... #
# # # _ _ _ _ _ _ # Permitir Otras Opciones, modificar con vacio o no
allow_bandwidth_input=no # _ _ _ _ _ _ # vacio para limitar ancho de banda de entrada para todas las fuentes en kbits/sec o no
allow_bandwidth_output=no # _ _ _ _ _ _ # vacio limitar ancho de banda de salida en kbits/sec por cada destino o no
allow_ping_send=no # _ _ _ _ _ _ # vacio para permitir enviar ping o no
allow_ping_recive=no # _ _ _ _ _ _ # vacio para permitir recivir ping o no
allow_forwading=no # _ _ _ _ _ _ # vacio para permitir reenvio, y no para denegar reenvios
allow_nat=no # _ _ _ _ _ _ # vacio para hacer masquerade a otras redes o no
allow_vpn=no # _ _ _ _ _ _ # vacio para permitir hosts excepcionales o no
allow_blacklist=no # _ _ _ _ _ _ # vacio para denegar hosts excepcionales o no
allow_output_uid=no # _ _ _ _ _ _ # vacio permite salida de paquetes de un usuario excepcional o no
allow_output_gid=no # _ _ _ _ _ _ # vacio permite salida de paquetes de un grupo excepcional o no
allow_others_protocols=no # _ _ _ _ _ _ # vacio para permitir otro protocolo ip excepcional o no
# # # _ _ _ _ _ _ # Otras Opciones, añadir con , y poner rangos con :
bandwidth_input=12512 # _ _ _ _ _ _ # maximo entrada de ancho de banda en kbit/sec para todos
bandwidth_output=512 # _ _ _ _ _ _ # maximo salida de ancho de banda en kbits/sec para ip
vpn=wesnoth.org # _ _ _ _ _ _ # permitir estos host excepcionales
blacklist= # _ _ _ _ _ _ # deniega estos host excepcionales
output_uid=root # _ _ _ _ _ _ # permite salida de paquetes de este usuario excepcional
output_gid=root # _ _ _ _ _ _ # permite salida de paquetes de este grupo excepcional
others_protocols=icmp,igmp # _ _ _ _ _ _ # permitir otros protocolos desde /etc/protocols
# # # _ _ _ _ _ _ # Red para conectar como cliente o server
net_ipv4_client=0/0 # _ _ _ _ _ _ # la red cual cliente ipv4, todos es 0/0
net_ipv4_server=0/0 # _ _ _ _ _ _ # la red cual servidor ipv4, todos es 0/0
net_ipv6_client=::/0 # _ _ _ _ _ _ # la red cual cliente ipv6, todos es ::/0
net_ipv6_server=::/0 # _ _ _ _ _ _ # la red cual servidor ipv6, todos es ::/0
# # # _ _ _ _ _ _ # FINAL .......... Opciones opcionales .......... .......... #

 Notas de fwiptables :

################################################## [only_output]

     Con [only_output] las reglas del firewall son mas utiles

##################################################  [vpn]

     Host [vpn] excepcionales tienen conexion permitida en el firewall

##################################################  [blacklist]

     Host [blacklist] excepcionales tienen conexion denegada en el firewall

##################################################  [uid y/o gid excepcional]

     Usuario y/o grupo excepcional tienen conexion de salida permitida en el firewall

##################################################  [Dos iptables netftilter]

     iptables-legacy   es    soporte para xtables 
     iptables-nft      es    soporte para neftables 

##################################################  [Puertos cliente udp necesarios]

     necesario para actualizar la fecha :   ntp
     necesario para dhcp y conseguir ip :   bootpc
     necesario para resolver dominios   :   domain

##################################################  [Protocolos necesarios para ipv6]

     Necesario protocolo: ipv6-icmp
     Ipv6 funciona tambien con: ipv4 procolos
